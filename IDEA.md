# Context Switcher — 개선 아이디어

> 작성일: 2026-03-09
> 디자인 기준: Apple iOS 26 스타일 (Liquid Glass, SF Pro, 스프링 애니메이션)

---

## 1. UX 개선

### 1-1. 레이아웃 & 내비게이션
- **슬라이딩 사이드바 패널**: 현재 고정된 사이드바를 스와이프/토글로 접을 수 있게 해 에디터 영역을 풀스크린으로 활용
- **탭바 리네임**: 더블클릭으로 폴더 탭의 이름(홈/업무 등)도 직접 편집 가능하게
- **탭 추가/삭제**: `+` 버튼으로 새 폴더 탭 생성, 길게 눌러 탭 삭제 (현재 4개 고정)
- **분할 비율 조절**: 에디터 1/2 사이 구분선을 드래그해 비율 조절 (현재 50:50 고정)
- **에디터 3분할 옵션**: 1개 또는 2개 에디터 레이아웃 토글 버튼
- **키보드 단축키**:
  - `Cmd+1~4`: 폴더 탭 전환
  - `Cmd+N`: 새 아이템 생성 (입력창 포커스)
  - `Cmd+W`: 현재 에디터 닫기
  - `Cmd+P`: 빠른 파일 검색(Quick Open)

### 1-2. 아이템 리스트
- **인라인 미리보기**: 아이템 호버 시 내용 첫 줄을 툴팁 또는 말풍선으로 표시
- **태그/색상 레이블**: 아이템에 색상 점(•) 또는 태그를 붙여 시각적 구분
- **즐겨찾기 고정**: 별표 아이콘으로 즐겨찾기 등록, 리스트 최상단 고정
- **일괄 선택 & 이동**: Shift+클릭 다중 선택 후 다른 폴더로 일괄 이동/아카이브
- **검색 필터**: 사이드바 상단에 인라인 검색창 — 입력 즉시 리스트 필터링

### 1-3. 에디터
- **마크다운 프리뷰 토글**: `Cmd+Shift+P`로 편집/미리보기 전환 (marked.umd.js 이미 포함)
- **글자 수 / 줄 수 표시**: 에디터 하단 상태바에 현재 커서 위치 및 통계
- **포커스 모드**: 현재 에디터만 전체화면으로 확대, 나머지 UI 페이드아웃
- **자동 저장 인디케이터 개선**: '저장 중...' 대신 iOS 스타일 작은 스피너 애니메이션

### 1-4. 아카이브
- **아카이브 내 검색**: 모달 상단 검색창으로 아카이브 파일 필터링
- **아카이브 날짜 표시**: 아카이브된 날짜를 보조 텍스트로 표시
- **영구 삭제 옵션**: 아카이브 항목에 삭제(🗑) 버튼 추가 (확인 다이얼로그 포함)

### 1-5. 데일리 메모 (Memo 탭)
- **날짜 네비게이션**: 좌/우 화살표로 이전/다음 날짜 파일로 이동
- **주간 캘린더 뷰**: 메모 탭 상단에 이번 주 날짜 버튼 표시, 해당 날 메모 바로 열기
- **스트릭 표시**: 연속으로 메모를 작성한 날수(streak) 배지

## 2. 기능 개선

### 2-1. 파일 & 데이터
- **파일 형식 확장**: `.txt` 외 `.md` 확장자 지원, 마크다운 에디터로 자동 전환
- **버전 히스토리**: 파일 저장 시 `.git`-lite 방식으로 스냅샷 보존 → 특정 시점으로 복원
- **내보내기 / 가져오기**: 폴더 전체를 ZIP으로 내보내기, ZIP/폴더 가져오기
- **iCloud / 로컬 동기화 경로 설정**: userData 외 사용자 지정 경로에 저장 가능

### 2-2. 인터랙션
- **빠른 캡처 (Quick Capture)**: 글로벌 단축키(`Cmd+Shift+Space`)로 미니 입력창 팝업 — 메모/아이디어를 바로 특정 폴더에 저장
- **드래그 & 드롭 외부 파일**: OS에서 텍스트 파일을 앱으로 드래그해 임포트
- **다중 창 지원**: 에디터 패인을 별도 윈도우로 분리(detach)

### 2-3. 시각화
- **폴더별 아이템 개수 뱃지**: 탭 옆에 숫자 뱃지 표시
- **히트맵 캘린더**: 메모 탭에서 GitHub 잔디 스타일로 작성 빈도 시각화
- **타임라인 뷰**: 아이템을 생성일 기준 타임라인으로 정렬해 보기

## 3. AI / LLM 접목 아이디어

> Claude API (`claude-sonnet-4-6` 권장) 사용 기준으로 작성

### 3-1. AI 글쓰기 어시스턴트 (에디터 내)
- **인라인 AI 패널**: 에디터 오른쪽 슬라이딩 패널로 AI 채팅창 표시
- **선택 텍스트 액션**: 텍스트 드래그 → 컨텍스트 메뉴에 AI 옵션
  - `요약`: 선택한 내용을 2~3줄로 요약
  - `다듬기`: 문장을 더 명확하고 자연스럽게 개선
  - `번역`: 한↔영 토글 번역
  - `계속 작성`: 커서 이후 내용을 AI가 이어서 작성
  - `아이디어 확장`: 짧은 메모를 바탕으로 세부 아이디어 제안

### 3-2. 스마트 자동완성
- **제목 자동 제안**: 아이템 입력창에서 내용을 일부 입력하면 AI가 제목 후보 3개 제안
- **연관 아이템 추천**: 현재 열린 파일과 의미적으로 유사한 다른 파일 링크 표시

### 3-3. 지식 관리 AI
- **전체 노트 검색 (Semantic Search)**: 키워드가 아닌 의미 기반 검색 — "지난주에 쓴 마케팅 관련 아이디어" 같은 자연어 질의
  - 구현: 각 파일 저장 시 임베딩 생성 후 로컬 벡터 인덱스에 저장

- **아이디어 클러스터링**: `idea` 폴더의 아이템들을 AI가 자동 그룹화하고 라벨 부여

### 3-4. 컨텍스트 인식 기능
- **스마트 아카이브 제안**: AI가 오랫동안 열지 않은 파일을 분석해 "이 파일들을 아카이브할까요?" 알림
- **중복 감지**: 새 아이템 생성 시 기존 파일 중 유사한 내용이 있으면 경고 표시
- **태그 자동 생성**: 파일 내용을 분석해 태그/카테고리 자동 제안

### 3-5. AI 채팅 플로팅 버튼

> ⚠️ **컨텍스트 전략 중요**: 전체 노트를 매번 LLM에 전송하면 토큰 낭비가 심하고 사용자가 원치 않는 내용이 외부 서버로 전송될 수 있음. 아래 전략 중 조합해서 적용할 것.

**컨텍스트 전달 전략 (비교)**

- **전략 A — 열린 파일만 전송 (기본값, 권장)**
  - 현재 에디터에 열려 있는 파일(최대 2개)만 LLM에 전송
  - 사용자가 명확히 "열어놓은 것"만 가므로 프라이버시 침해 없음
  - 토큰 최소화, UX도 직관적
  - 질문 예시: "이 파일 내용을 요약해줘", "이걸 바탕으로 이메일 초안 써줘"

- **전략 B — `@파일명` 으로 명시적 추가**
  - 채팅창에서 `@업무메모` 처럼 타이핑하면 해당 파일 내용을 컨텍스트에 추가
  - 사용자가 의도적으로 선택한 파일만 전송 → 프라이버시 통제권 사용자에게 부여
  - 전송 전 "이 파일들을 AI에게 전송합니다" 배지 표시

- **전략 C — RAG (Retrieval-Augmented Generation, 중장기)**
  - 질문 입력 시 앱 내부에서 의미적으로 관련된 파일 2~3개만 자동 선택 후 전송
  - 전체 노트를 보내지 않고도 "전체 노트에서 검색"하는 효과
  - 구현: 파일 저장 시 로컬 임베딩 생성 → 질문 시 유사도 검색 → 상위 N개만 전송
  - 임베딩 자체는 로컬 모델(예: `nomic-embed-text` via Ollama) 또는 별도 임베딩 API 사용

- **전략 D — 로컬 LLM (완전 오프라인)**
  - Ollama 연동 시 모든 데이터가 로컬에서만 처리 → 프라이버시 완벽
  - 단, 모델 품질이 클라우드 LLM보다 낮고 초기 설치 허들 있음

**플로팅 버튼 UX**
- 우하단 플로팅 AI 버튼, 클릭 시 채팅 패널 슬라이드업
- 기본 컨텍스트는 전략 A (열린 파일), `@멘션`으로 추가 가능 (전략 B)
- 채팅창 상단에 "현재 AI가 참고 중인 파일" 칩(chip) 표시 — 투명성 확보
- 질문 예시: "이 파일 요약해줘", "이번 주 업무 메모 보고서로 만들어줘"

### 3-6. 구현 고려사항
- **API 키 관리**: 설정 화면에서 Anthropic API Key 입력, 로컬 OS Keychain에 저장
- **프라이버시 원칙**: AI에게 전송되는 파일 목록을 항상 UI에 표시, 사용자가 명확히 인지하도록
- **AI 기능 on/off 토글**: 민감 데이터 우려 사용자를 위해 AI 기능 전체 비활성화 옵션
- **로컬 LLM 옵션**: Ollama 연동 옵션 제공 — 전략 D, 완전 오프라인 사용 가능
- **스트리밍 응답**: 긴 생성 결과는 스트리밍으로 타이핑 효과 표시
- **토큰 사용량 표시**: 설정 화면에서 이번 달 누적 토큰 사용량 확인
- **컨텍스트 크기 제한**: 파일 크기가 클 경우 자동으로 앞부분 N토큰만 잘라서 전송, 사용자에게 알림
